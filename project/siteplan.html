A practical hub of hiking routes, packing tips, and community-sourced trail reviews.

2. Site Name

TrailTrek

Why: Memorable, descriptive, and conveys hiking/trail exploration. Works across devices and fits available domain possibilities (e.g. trailtrek.org — domain availability should be checked prior to purchase).

3. Site Purpose

The purpose of TrailTrek is to provide concise, trustworthy hiking trail information for day hikers and weekend backpackers. The site will include:

  • Trail summaries (distance, difficulty, elevation gain, user reviews).
  • Packing & safety checklists and printable PDFs.
  • Interactive trail finder (filter by difficulty, distance, region).
  • Contact / volunteer sign-up form and an external resources & citations page.

4. Scenarios

  1. New hiker: "I have 3 hours and want a beginner-friendly trail near me — what should I pick?"
  2. Photographer: "Which trails have viewpoints at sunrise and low crowd levels?"
  3. Volunteer: "How do I sign up to help maintain trails in my county?"

These questions drive content: trail metadata, searchable filters, and a clear contact/volunteer form.

5. Color Schema

Two main colors + neutrals. Swatches below are used throughout this document.

  • Primary — Forest Green #2E8B57 (Headings, primary CTAs, focus states)
  • Accent — Amber #F4A300 (Buttons, highlights, icons)
  • Neutral — Off-white background #FBFBFA and dark text #222222

Usage examples: headings use #2E8B57, primary buttons use a filled #F4A300, page background uses #FBFBFA.

6. Typography

Selected web fonts (Google Fonts):

  • Montserrat — headings and UI labels (weights 600–700)
  • Inter — body text and form inputs (weights 300–400)

These are used in this document (see header and body fonts). Limit of two fonts to maintain performance and consistency.

8. CSS / Accessibility / Testing

  • Validate HTML at validator.w3.org/nu and CSS at jigsaw.w3.org/css-validator.
  • Use DevTools Lighthouse for Performance, Accessibility, Best Practices, and SEO (mobile & desktop).
  • Check color contrast using DevTools CSS Overview or a contrast tool to ensure 4.5:1 for body text vs background.
  • Optimize images (compress and serve responsive sizes) and use loading="lazy" for non-critical images.
  • JS requirements (final project): multiple functions, DOM selection/modification, event listeners, conditional branching, arrays/objects and methods, template literals, localStorage — all will be implemented in the final site pages.
